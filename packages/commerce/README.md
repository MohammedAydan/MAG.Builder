# @nexpress/commerce

Typed commerce boundary for NexPress.

Phase 16 adds:

- provider-agnostic adapter contracts for products, prices, carts, customers, and orders
- lazy runtime config validation for commerce providers
- a Medusa adapter skeleton with connectivity smoke checks
- an in-memory adapter for tests and contract verification

Phase 16 does not add storefront UI, checkout, payments, shipping, taxes, inventory, or order workflows.
