# Phase 01 Plan

## Goal

Lock the v1 product and architecture decisions for NexPress, make scope boundaries explicit, and record them in ADRs and governance documents without implementing platform features.

## Acceptance Criteria

- ADR files exist in the repository decision directory
- v1 scope is explicit and frozen
- out-of-scope items are explicit
- no decisions contradict `PLAN.md`, `01-final-decision-record.md`, or the architecture docs
- `IMPLEMENTATION_STATUS.md`, `plans/context.md`, and `plans/SESSION_LOG.md` reflect the Phase 01 outcome

## Approach

Create the Phase 01 planning files, mark the phase in progress, review the existing decision directory, add focused ADRs for v1 scope and architecture constraints, create a product scope document, then update status and review artifacts after verification.

## Scope

In scope: ADRs, scope freeze, governance docs, and status updates.

Out of scope: application code, framework integration, CMS setup, auth, admin, commerce, builder runtime, themes, templates, plugins, APIs, or MCP implementation.

## Dependencies

- `PLAN.md`
- `01-final-decision-record.md`
- `03-phases/README.md`
- `03-phases/phase-01-product-lock-and-adr/*`
- relevant files under `02-architecture/`
