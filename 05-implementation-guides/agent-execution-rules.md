# Agent Execution Rules

1. Never implement more than the requested phase.
2. Always update `IMPLEMENTATION_STATUS.md`.
3. Always report commands run and failures.
4. Never hide failing tests.
5. Never add arbitrary plugin runtime execution.
6. Never expose secrets in logs or examples.
7. Prefer official docs when installing or configuring fast-moving packages.
8. Any architecture change requires an ADR in `docs/decisions/`.
